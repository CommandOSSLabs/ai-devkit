# Feature Spec Template (Portable Baseline)

Use these sections in order unless the target repository has a stronger local convention.

## 1) Overview

- Problem statement, target users, intended outcome.

## 2) Design Principles

- Principle-by-principle requirements and rationale.
- Include both functional and non-functional requirements.

## 3) Data Model (Optional)

- Key entities and relationships relevant to the feature.

## 4) Flows

- Success and failure paths for each core flow.
- Include sequence diagram(s) when useful.

## 5) Boundaries (Optional)

- What this feature owns and does not own.
- Adjacent specs and integration points.

## 6) Technical Decisions

- Feature-scoped implementation choices and trade-offs.
- Link to ADRs for system-wide decisions.

## 7) Changelog

- Date-stamped summary of significant spec updates.

## 8) Open Points (Optional)

- Unresolved decisions and decision options.

## 9) Known Issues (Optional)

- Current gaps, impacts, and planned resolution direction.

## 10) Notes and References (Optional)

- Key file paths, PRs, diagrams, and external links.
